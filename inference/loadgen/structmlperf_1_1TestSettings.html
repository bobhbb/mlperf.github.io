Test duration

The test runs until both min duration and min query count have been met. However, it will exit before that point if either max duration or max query count have been reached.

Performance Sample modifiers

These settings can be used to Audit Performance mode runs. In order to detect sample caching by SUT, performance of runs when only unique queries (with non-repeated samples) are issued can be compared with that when the same query is repeatedly issued.

◆ server_target_latency_percentile

double mlperf::TestSettings::server_target_latency_percentile = 0.99

The latency percentile for server mode. This value is combined with server_target_latency_ns to determine if a run is valid.

99% is the default value, which is correct for image models. GNMT should be set to 0.97 (97%) in v0.5.(As always, check the policy page for updated values for the benchmark you are running.)
