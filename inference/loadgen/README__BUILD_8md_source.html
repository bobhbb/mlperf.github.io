9 ## Quick Start
10 
11  pip install absl-py numpy
12  git clone --recurse-submodules https://github.com/mlperf/inference.git mlperf_inference
13  cd mlperf_inference/loadgen
14  CFLAGS="-std=c++14 -O3" python setup.py bdist_wheel
15  pip install --force-reinstall dist/mlperf_loadgen-0.5a0-cp36-cp36m-linux_x86_64.whl
16  python demos/py_demo_single_stream.py
17 
18 This will fetch the loadgen source, build and install the loadgen as a python module, and run a simple end-to-end demo. The exact *.whl filename may differ on your system, but there should only be one resulting whl file for you to use.
19 
20 A summary of the test results can be found in the *"mlperf_log_summary.txt"* logfile.
21 
22 For a timeline visualization of what happened during the test, open the *"mlperf_log_trace.json"* file in Chrome:
23 * Type “chrome://tracing” in the address bar, then drag-n-drop the json.
24 * This may be useful for SUT performance tuning and understanding + debugging the loadgen.
25

41 ## Overview
42 
43 The load generator is built using the
44 [gn metabuild](https://gn.googlesource.com/gn/+/master)
45 and [ninja build](https://ninja-build.org/) tools.
46 
47 Using git submodules to manage a checkout will compile gn and ninja if needed.
48 Using depot\_tools to manage a checkout will include prebuilt versions of gn and ninja.
49 By default, both ways will use the system C++ compiler, however depot\_tools also
50 has the option to use versioned compiler binaries.
51 
52 If the tools above don't cover your particular configuration, please reach out.
53 Patches to support other build environments welcome.

55 ## Git Submodules Approach
56 
57 ### Downloading the source
58 
59 Download the mlperf inference repository and its submodules.
60 
61  git clone --recurse-submodules https://github.com/mlperf/inference.git

63 ### Build from source
64 
65 #### Just building once
66 
67 The following is a phony Makefile target, wrapping everything needed to build
68 the load generator. It'll get the job done, but will result in lots of redundant
69 work if used over and over again, so isn't recommended for development.
70 The resulting binary will be found in: out/MakefileGnProj/obj/loadgen/libmlperf\_loadgen.*
71 
72  make mlperf_loadgen
73 
74 To build the python module:
75 
76  make mlperf_loadgen_pymodule
77 
78 Note: These make targets assume Unix-like shell commands are available.
79 
80 #### Building for development purposes
81 
82 The bootstrap\_gn\_ninja make target will build ninja and gn into the current
83 tree and should only need to be called once per checkout.
84 This is not necessary if you already have gn and ninja installed on your
85 system.
86 
87  make bootstrap_gn_ninja
88 
89 Then generate the ninja build files from the gn build files.
90 You can have multiple sets of ninja build files in different out directories,
91 each with their own args. Release and debug sets, for example:
92 
93  third_party/gn/gn gen out/Release --args="is_debug=false"
94  third_party/gn/gn gen out/Debug --args="is_debug=true"
95 
96 From here you can edit+build over and over using one of the following targets:
97 
98  third_party/ninja/ninja -C out/Release mlperf_loadgen
99  third_party/ninja/ninja -C out/Release loadgen_pymodule_wheel_src
100 
101 You will find the binary in *"out/Release/obj/loadgen/libmlperf\_loadgen.a"*. (Or as a .lib on Windows).
102 
103 Link that library directly into the executable you want to test.
104 
105 Optionally, create a project file for your favorite IDE.
106 See [gn documentation](https://gn.googlesource.com/gn/+/master/docs/reference.md#ide-options) for details.
107 
108  gn gen --root=src --ide=eclipse out/Release
109  gn gen --root=src --ide=vs out/Release
110  gn gen --root=src --ide=xcode out/Release
111  gn gen --root=src --ide=qtcreator out/Release

113 ## Depot Tools approach
114 
115 ### Downloading the source
116 
117 Download and install depot\_tools:
118 
119  git clone 'https://chromium.googlesource.com/chromium/tools/depot_tools.git'
120  export PATH="${PWD}/depot_tools:${PATH}"
121 
122 Copy the depot\_tools fetch config for the MLPerf load generator to the
123 depot\_tools path:
124 
125  wget https://raw.githubusercontent.com/mlperf/inference/master/loadgen/depot_tools/fetch_configs/mlperf_loadgen.py \
126  -O ${PWD}/depot_tools/fetch_configs/mlperf_loadgen.py
127 
128 Create a folder for the load generator project and fetch the source code:
129 
130  mkdir mlperf_loadgen
131  cd mlperf_loadgen
132  fetch mlperf_loadgen
133  gclient sync
134 
135 ### Building from source
136 
137 The depot\_tools approach enables building with versioned toolchains.
138 
139 <i>Note: This has only been tested to work in a Debian-based linux environment so
140 far, but it should support many others.</i>
141 
142 Run the gn metabuild. The output directory will contain ninja build files for a
143 specific target platform and set of build options.
144 
145  gn gen --root=src out/Default
146 
147 TODO: Provide gn commands for cross-compiling to Android and to iOS. In the mean
148 time, looking at how to build chromium
149 [for android](https://chromium.googlesource.com/chromium/src/+/master/docs/android_build_instructions.md)
150 or [for ios](https://chromium.googlesource.com/chromium/src/+/HEAD/docs/ios/build_instructions.md)
151 should provide some useful pointers.
152 
153 Build the library:
154 
155  ninja -C out/Default mlperf_loadgen
156 
157 You will find the binary in *"out/Release/obj/loadgen/libmlperf\_loadgen.a"*. (Or as a .lib on Windows).
158 
159 Link that library directly into the executable you want to test.

161 ## Notes about important dependency and build files
162 
163 **DEPS**: Describes source repositories to pull in as dependencies depot\_tools'
164 "gclient sync" command. Also runs system commands to download the relevant toolchains
165 for the "gclient runhooks" command, which is run as part of the initial "fetch".
166 
167 **.gitmodules**: Git submodules alternative to DEPS.
168 
169 **.gn**: The root gn build file.
170 
171 **BUILD.gn**: Located in multiple directories. Each one describes how to build
172 that particular directory.
