3 ## Q: Can I make local modifications to the LoadGen for submission?
4 **A:** No. To keep the playing field level, please upstream any local
5 modificiations you need to make.
6 
7 ## Q: Where is version_generated.cc? I'm getting linker errors for LoadgenVersion definitions.
8 **A:** If you have a custom build setup, make sure you run the
9 version_generator.py script, which will create the cc file you are looking for.
10 The official build files that come with the LoadGen do this automatically for
11 you.
12 
13 ## Q: What is this version_generator.py script?
14 **A:** The LoadGen records git stats and the SHA1 of all its source files at
15 build time for verification purposes. This is easy to circumvent, but try your
16 best to run version_generator.py correctly; ideally integrated with your build
17 system if you have a custom build. The intention is more to help with debugging
18 efforts and detect accidental version missmatches than to detect bad actors.

20 ## Q: How do I view the mlperf_log_trace.json file?
21 **A:** This file uses the [Trace Event Format]
22 (https://docs.google.com/document/d/1CvAClvFfyA5R-PhYUmn5OOQtYMH4h6I0nSsKchNAySU/edit)
23 to record a timeline of all the threads involved.
24 You can view the file by typing [chrome://tracing](chrome://tracing) into
25 Chrome's address bar and dragging the json file there.
26 This file zips well and you can drag the zip file directly into
27 "chrome:tracing" too.
28 Please include zipped traces (and the other logs) when filing bug reports.
29 
30 ## Q: What is the difference between the MultiStream and MultiStreamFree scenarios?
31 **A:** MultiStream corresponds to the official MLPerf scenario for submissions;
32 it has a fixed query rate and allows only one outstanding query at a time.
33 MultiStreamFree is implemented for evaluation purposes only; it sends queries
34 as fast as possible and allows up to N outstanding queries at a time. You may
35 want to use MultiStreamFree for development purposes since small improvements
36 in performance will always be reflected in the results, whereas MultiStream's
37 results will be quantized.
