Detailed Description

Implements a logging system with a central IO thread that handles all stringification and IO.

Log-producing threads only submit lambdas to be executed on the IO thread. All producers and consumers use lock-free operations that guarantee forward progress independent of a) other stalled threads and b) where those threads are stalled. Each thread uses a double-buffering scheme to queue its logs. One buffer is always reserved for writes and the other is reserved for reads. A producing thread sends requests to the IOThread to swap the buffers and the IOThread does the actual read/write swap after it has finished reading the buffer it was working on.
